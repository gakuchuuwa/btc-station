**版本**: v2.0
**日期**: 2026-04

---

## 一、 项目背景与终极目的 (The "WHY")

### 1. 核心目的
本项目旨在将现有的本地化单点量化交易工具升级为一个**面向多用户的专业量化交易 SaaS 平台**。平台将为用户提供从数据查看、策略回测、参数寻优到实盘部署的完整闭环服务，核心差异化优势是“支持 Python 级专业策略开发”。

### 2. 为什么升级为 SaaS？
* **扩大受众**：不再局限于本地开发者使用，让普通客户（无编程环境）也能通过浏览器直接享受量化投研服务。
* **数据集中化**：避免多客户端同时拉取数据导致 API 封禁，实现服务器统一拉取、内存级高速分发。
* **策略变现**：未来可通过售卖内置策略模板或算力服务实现商业化。

### 3. 平台核心愿景
打造一个“Web 版的量化实验室”。首页提供实时新闻与大盘概览，工作站页面提供类似 TradingView 的专业图表与极速的回测引擎。

## 二、 核心功能规划 (The "WHAT")

> [!IMPORTANT]
> 核心原则：做减法。因为我们只做 BTC，抛弃了跨币种、跨市场的复杂逻辑，使得系统可以做到极速和极简。

### 模块 A：数据基座 (Data Engine)
* **功能**：通过 CCXT 直连交易所，自动拉取 BTC 各周期的历史 K 线数据。
* **特性**：本地 SQLite/Parquet 缓存，拒绝网络延迟，实现“毫秒级”读取。

### 模块 B：极速回测大脑 (Backtest Core)
* **功能**：基于 `VectorBT` 库重写海龟交易法/均线等策略。
* **特性**：利用底层的 Numpy 矩阵运算，实现成千上万次参数组合在**秒级**内完成测算。

### 模块 C：全景驾驶舱 (Frontend Cockpit)
* **功能**：一个基于浏览器的前端页面。
* **特性**：
  * 内嵌 `TradingView Lightweight Charts` 实现原汁原味的 TV 级看盘体验。
  * 提供 3D 热力图直观展示参数优化的结果，找出“盈利高原区”。

### 模块 D：无人值守实盘引擎 (Live Execution)
* **功能**：参数调优后，一键切换至“实盘模式”。
* **特性**：系统后台守护进程，7x24小时监控实时价格并利用 CCXT 执行买卖操作，接入 Telegram 进行报表推送。

## 三、 技术选型与架构 (The "HOW")

| 组件层级 | 技术栈选择 | 选择原因 |
| :--- | :--- | :--- |
| **前端 (UI/UX)** | React + Vite + TailwindCSS | 组件化开发，生态强大，开发效率极高。 |
| **图表渲染** | Lightweight Charts + Plotly | 前者完美复刻 TV 体验，后者用于复杂 3D 参数矩阵可视化。 |
| **后端 API** | Python + FastAPI | 极高并发，天生支持异步，Python 量化生态的首选。 |
| **量化核心** | VectorBT + Pandas | 性能远超普通回测框架，专为大规模参数优化而生。 |
| **交易网关** | CCXT | 全球公认最稳定的加密货币通用交易所接入库。 |

### 部署架构 (Deployment)
* **前端 (Client)**: 编译为纯静态文件，部署到 GitHub Pages 并绑定你的专属域名 `https://quant-lab.org/`。
* **后端 (Server)**: 部署到独立的云服务器（如阿里云、AWS EC2 等）。因为强大的 `VectorBT` 矩阵运算引擎和 CCXT 数据抓取**必须**运行在真正的 Python 服务器上，无法在 GitHub Pages（仅支持静态网页）中运行。
* **通信方式**：`quant-lab.org` 的网页通过跨域请求 (CORS) 调用你的云服务器 IP/域名 获取回测计算结果。

## 四、 执行路线图 (Roadmap)

此计划严格遵循 **RALPH LOOP** 迭代原则，将庞大的系统拆分为5个绝对可控的阶段：

* [x] **Phase 1: 数据与底层基座搭建** (当前正在进行：完成 FastAPI 后端与 CCXT 数据抓取)。
* [ ] **Phase 2: 回测引擎整合** (将策略逻辑注入 VectorBT 并暴露接口)。
* [ ] **Phase 3: 前端驾驶舱搭建** (启动 React 项目，画出能标记买卖点的 K 线图)。
* [ ] **Phase 4: 超参数暴力寻优台** (实现网页端直接输入区间，返回 3D 胜率图表)。
* [ ] **Phase 5: 自动实盘网关** (编写后台异步发单逻辑与风控系统)。
